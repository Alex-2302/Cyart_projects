# Rules of Engagement (RoE)

## Purpose

The purpose of this engagement is to perform a controlled Red Team assessment to identify security weaknesses in a target system and demonstrate potential attack scenarios in a safe and ethical manner.

---

## Scope

- Target System: Metasploitable2 Virtual Machine  
- Environment: Isolated lab setup  
- Network: Local virtual network only  

Only the specified system within the lab environment is included in the scope.

---

## Out of Scope

- Any systems outside the lab environment  
- Real-world networks or devices  
- Denial-of-Service (DoS) attacks  
- Data destruction or modification  

---

## Allowed Activities

- Network scanning and reconnaissance  
- Vulnerability assessment  
- Exploitation of known vulnerabilities  
- Post-exploitation techniques (non-destructive)  
- Password testing using controlled methods  

---

## Restrictions

- No permanent changes to the system  
- No deletion or corruption of data  
- No disruption of system availability  
- Activities must remain within ethical boundaries  

---

## Data Handling

- No sensitive data will be extracted or stored  
- Any collected information will be used only for educational purposes  
- Results will remain confidential within the lab context  

---

## Authorization

All activities are performed in a controlled lab environment with permission for educational purposes. No unauthorized systems are targeted.

---

## Reporting

All findings will be documented, including vulnerabilities, attack paths, and recommendations. The report will focus on improving system security.

---

## Conclusion

This engagement is conducted to simulate real-world attack scenarios while maintaining safety and ethical standards. The goal is to enhance understanding of system vulnerabilities and improve defensive measures.
